## 三、实验任务与步骤（记录并粘贴输出）

### 任务一：MAC 地址学习过程
1. 按照拓扑接线，设置主机 IP，确认交换机为初始状态。
2. 在 PCA 上执行 `ping PCC`，记录交换机 `display mac-address` 输出（粘贴截图）。
3. 分析 MAC 表，说明每个表项的 VLAN、端口与学习来源。

结果占位：
- 在 `jianceA` 上学到的 MAC：`000f-e2f3-5bc7` -> Ethernet1/0/24；`0021-70fe-a294` -> Ethernet1/0/1；……（粘贴实际输出）

### 任务二：Access 链路端口配置
1. 查看默认 VLAN (`display vlan all`) 与接口简要信息 (`display brief interface`)，记录 PVID 与端口类型。
2. 在 `jianceA`/`jianceB` 上创建 `vlan 10` 与 `vlan 20`，并将端口分配到相应 VLAN（示例命令见实验指导书）。
3. 验证 VLAN 划分：`display vlan all` 并粘贴输出。
4. 连通性测试：设置 PCA 属于 VLAN10，PCB 属于 VLAN20，PCA ping PCB（记录超时/不可达截图）。

结论占位：不同 VLAN 之间默认不能互通；若交换机间接入端口未设 trunk，则相同 VLAN 的主机跨交换机也可能不可达（粘贴解释）。

### 任务三：Trunk 链路端口配置
1. 配置交换机间 uplink 端口 `Ethernet1/0/24` 为 trunk：
   - `interface Ethernet 1/0/24`
   - `port link-type trunk`
   - `port trunk permit vlan all`
2. 查看端口与 VLAN 关联：`display interface Ethernet 1/0/24` 与 `display vlan all`（粘贴输出）。
3. 重新测试 PCA 与 PCC（跨交换机同 VLAN），记录 `ping` 成功截图。

结论占位：配置 trunk 后，同一 VLAN 的主机可跨交换机互通，因 trunk 传递带标签的 VLAN 帧。
